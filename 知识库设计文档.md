# 个人知识库设计文档

> 2026-05-09 · VuePress 2 + GitHub Pages · 半公开知识管理系统

---

## 1. 项目概述

### 1.1 定位

一个基于 **VuePress 2 + GitHub Pages** 的个人知识库，以 Markdown 为核心，Git 为版本管理工具，兼顾个人知识管理效率与半公开分享需求。

### 1.2 核心需求

| 维度 | 决策 |
|------|------|
| 内容类型 | 综合：技术笔记 + 读书笔记 + 思考复盘 + 生活记录 |
| 公开程度 | 半公开：自用为主，部分可分享，允许搜索引擎收录 |
| 写作方式 | 技术内容纯 Markdown，非技术内容 Obsidian WYSIWYG |
| 组织方式 | 文件夹层级（主干） + 标签（纽带） |
| 全文搜索 | 刚需，必须支持中文分词 |
| 部署 | GitHub Actions 自动构建 → GitHub Pages |

### 1.3 框架选型

**选择 VuePress 2**，对比理由：

| 维度 | Docsify | VuePress 2 | Hexo |
|------|---------|------------|------|
| Markdown 原生 | ★★★ 直接渲染 | ★★☆ frontmatter + .md | ★★☆ 约定较多 |
| 文件夹组织 | ★★★ 完全自由 | ★★★ 文件夹即路由 | ★★☆ _posts 约定 |
| 全文搜索 | ★★☆ 客户端分词 | ★★★ 预建索引 | ★★★ 预建索引 |
| SEO | ★☆☆ CSR 渲染 | ★★★ SSG 预渲染 | ★★★ SSG 预渲染 |
| 标签系统 | ★☆☆ 手写 | ★★★ 内置支持 | ★★★ 原生支持 |
| 部署复杂度 | ★★★ 即推即用 | ★★☆ 一次配置 | ★★☆ 需要构建 |
| 生态/插件 | ★★☆ | ★★★ | ★★★ |

**结论**：VuePress 2 在 SEO + 全文搜索 + 标签系统 + 文件夹自由组织这四个核心需求上表现最均衡。


## 3. 内容规范

### 3.1 Frontmatter 字段规范

每篇 `.md` 文件必须包含 YAML frontmatter：

```yaml
---
title: React 18 并发特性深入理解     # 必填，文章标题
date: 2026-05-09                     # 必填，创建日期
category: 前端                        # 必填，分类（对应顶层文件夹）
tags:                                 # 推荐，标签数组
  - React
  - 并发
  - 性能优化
sticky: false                         # 可选，是否置顶，默认 false
draft: false                          # 可选，是否草稿，默认 false
---
```

### 3.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 是 | 文章标题，显示在页面和搜索结果中 |
| `date` | YYYY-MM-DD | 是 | 创建日期，用于时间线排序 |
| `category` | string | 是 | 主分类，对应顶层文件夹 |
| `tags` | string[] | 推荐 | 标签列表，用于跨域聚合 |
| `sticky` | boolean | 否 | 置顶标记，默认 false |
| `draft` | boolean | 否 | 草稿标记，true 时构建跳过 |

### 3.3 组织逻辑

- **文件夹** = 文章的主要归属地，决定 URL 路径（如 `tech/frontend/react-concurrency.md`）
- **category** = 与顶层文件夹对应，用于侧边栏分组
- **tags** = 跨文件夹的概念关联，自动生成标签聚合页
- **一篇文章** 同时出现在文件夹侧边栏 + 每个标签的聚合页中

## 4. 功能设计

### 4.1 导航结构

```
┌─ 侧边栏（左）──────────────────┬─ 内容区 ────────────┐
│                                │                     │
│  🔍 搜索...                    │  文章标题            │
│                                │  2026-05-09          │
│  📁 技术笔记                    │  🏷️ React | 并发    │
│    ├─ 前端                      │                     │
│    ├─ 后端                      │  正文内容...         │
│    └─ DevOps                    │                     │
│  📁 读书笔记                    │                     │
│  📁 思考复盘                    │                     │
│  📁 生活记录                    │                     │
│                                │                     │
│  🏷️ 标签索引                   │                     │
│    React (12) | 性能 (8) ...   │                     │
└────────────────────────────────┴─────────────────────┘
```

### 4.2 全文搜索

- 使用 `@vuepress/plugin-search`
- 构建时从所有 `.md` 文件提取文本，生成 JSON 搜索索引
- 搜索范围：标题 + 正文内容
- 结果展示：标题 + 匹配片段高亮
- 中文分词：默认分词策略配合 nodejieba（可选，按需引入）

### 4.3 标签系统

- 构建时扫描所有 `.md` 文件的 `tags` 字段
- 自动生成 `/tags/` 下的标签聚合页
- 每个标签页列出所有关联文章，含所属文件夹和日期
- 标签页支持分页

### 4.4 首页设计

首页展示：
1. 最近更新的文章列表（按 date 倒排，排除 draft）
2. 标签云（字号按文章数量加权）
3. 分类卡片式入口（4 个顶级分类）

## 5. 写作工作流

### 5.1 双轨写作

| 场景 | 工具 | 模式 |
|------|------|------|
| 技术文章 | VS Code / Obsidian 源码模式 | 纯 .md 编辑，完全控制 |
| 非技术内容 | Obsidian 实时预览模式 | 所见即所得，降低写作心理门槛 |
| 紧急编辑 | GitHub 网页端 | 在线编辑 .md，不需要本地环境 |

### 5.2 发布流程

```
新建/编辑 .md → 设置 frontmatter
                   ↓
            draft: true          draft: false
                ↓                     ↓
         git push              git push
         → 构建时跳过           → 构建时包含
         → 本地 dev 可见        → 部署上线
         → 改 false 即发布
```

### 5.3 本地开发

```bash
pnpm install
pnpm docs:dev     # 启动本地开发服务器 http://localhost:8080
pnpm docs:build   # 构建静态站点到 docs/.vuepress/dist/
```

## 6. 部署方案

### 6.1 CI/CD 流程

触发条件：`push` 到 `main` 分支

```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - uses: pnpm/action-setup@v2
      - run: pnpm install
      - run: pnpm docs:build
      - uses: actions/deploy-pages@v4
```

### 6.2 分支策略

| 分支 | 用途 |
|------|------|
| `main` | 唯一源分支，所有内容提交到这里 |
| `gh-pages` | GitHub Actions 自动管理，不手动操作 |

### 6.3 域名

- **默认**：`<username>.github.io/<repo-name>`，零配置，免费 HTTPS
- **自定义域名**（可选）：在 `docs/.vuepress/public/CNAME` 中添加域名

## 7. 主题与外观

### 7.1 主题选择

使用 VuePress 2 默认主题（`@vuepress/theme-default`），理由：
- 与 VuePress 2 深度集成，无需额外配置
- 侧边栏 / 导航栏 / 搜索 / 标签功能开箱即用
- 响应式设计，移动端友好
- 支持自定义调色板和 CSS 变量覆盖

### 7.2 配色方案

```
浅色模式：
  背景: #ffffff
  侧边栏: #f8f8f8
  文字: #2c3e50
  链接: #3a5ccc
  代码块: #f0f0f0

深色模式（默认主题自动支持）：
  背景: #1e1e1e
  侧边栏: #252526
  文字: #e0e0e0
  链接: #6c8cff
```

### 7.3 视觉配置要点

- 侧边栏按文件夹层级自动折叠
- 文章内代码块使用 Prism.js 高亮（默认）
- 支持 mermaid 图表渲染（插件按需启用）
- 支持 LaTeX 数学公式（插件按需启用）

## 10. 设计决策记录

| 决策点 | 选择 | 核心理由 |
|--------|------|----------|
| 框架 | VuePress 2 | SEO + 搜索 + 标签三个刚需的最佳平衡点 |
| 不引入 CMS | 纯 .md 工作流 | 极简维护，Obsidian 已满足富文本需要 |
| 单分支 | main 唯一 | 个人项目，draft 机制已满足草稿需求 |
| 默认主题 | @vuepress/theme-default | 开箱即用，减少定制成本 |
| 搜索 | @vuepress/plugin-search | 预建索引，客户端搜索够用，不依赖外部服务 |
| 不引入 Algolia | 内置搜索 | 个人项目规模不需要，省去外部依赖 |
