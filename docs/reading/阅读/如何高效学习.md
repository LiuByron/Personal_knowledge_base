# 如何高效学习

最近读完了 Scott Young 的《如何高效学习》，结合自己的经验做了一些整理。

## 核心观点

学习的本质不是"存储信息"，而是**建立连接**。你脑子里的知识像一张网，新知识能挂靠在旧知识上，才算真正学会了。

## 费曼技巧

教给别人是验证理解的最好方式：

1. 拿出一张白纸，写下要学习的概念
2. 用最简单的语言解释它，假装讲给一个完全不懂的人听
3. 卡壳的地方就是你没真懂的地方——回头查资料
4. 重复以上步骤，直到能用一句话说清楚

把复杂的东西讲简单，才是真的懂了。

## 间隔重复

临时抱佛脚的知识忘得最快。Anki 这类间隔重复工具的本质是：在你快忘记的时候，恰好帮你复习一遍。记忆曲线被拉长了，遗忘被延缓了。

我现在把前端技术笔记整理到这个知识库里，本身也是一种"间隔复习"——写文章的过程强迫自己把理解组织成文字。

## 主动学习 > 被动输入

| 被动输入 | 主动学习 |
|----------|----------|
| 看视频教程 | 写代码实现 |
| 读文档 | 回答别人问题 |
| 听播客 | 做笔记 + 复述 |
| 收藏文章 | 写一篇总结 |

被动输入的问题在于**让你产生"我学会了"的错觉**。只有输出才能检验到底学会没有。

## 小步快跑

不要试图一口吃成胖子。学 React 别同时学 Redux、Next.js、TypeScript、Tailwind。先只学 React，做一个小项目，跑通了再加下一块积木。
