# AI 工具使用指南

## 一、Superpowers：社区公认的"开发全家桶"

如果说只装一个技能，那绝对是 **Superpowers**。它由社区大神 obra 开发，并不是单一功能，而是把 20 多个开发子技能打包成了一个强大的工程化框架。

**核心作用：** 它覆盖了从"想清楚要做什么"到"写完代码自己检查"的完整链路。

### 杀手级子技能

| 指令 | 作用 |
|------|------|
| `/superpowers:tdd` | 强制 AI 先写测试用例，再写业务代码，极大提升代码健壮性 |
| `/superpowers:debug` | 系统化排查 Bug，而不是盲目打补丁 |
| `/superpowers:code-review` | 在代码合并前自动进行审查，拦截潜在漏洞 |

**适用场景：** 任何严肃的后端、全栈或前端项目开发，它是保证 AI 不写"屎山代码"的底线。

## 二、Skill Creator：打造你专属的"元技能"

这是 Anthropic 官方出品的工具，也是最被低估的**"元技能"**。

**核心作用：** 它可以让你用自然语言描述自己的工作流，然后自动生成标准格式的 Skill 文件。比如你可以告诉它：

> "帮我创建一个技能，在每次 git commit 前自动检查是否有 console.log 残留"

它就能帮你把这个规范固化下来。

**适用场景：** 封装团队特有的编码规范、固化个人的调试流程、沉淀项目最佳实践。

## 三、Agent 工具平台对比

### 3.1 gstack：软件研发的"正规军"

#### 优势

- **极致的工程化流程：** gstack 把 YC 总裁 Garry Tan 20 年的产品研发经验固化成了一套标准作业程序（SOP）。它强制要求从需求评审、架构设计、代码审查到测试上线，每个环节都有对应的 AI 角色（CEO、工程经理、QA、安全官等）进行把关，从根源上杜绝"半成品"代码和技术债。
- **硬核的自动化测试：** 它拥有业内顶尖的真实浏览器测试能力。通过在后台常驻浏览器进程，AI 可以保留登录态（直接导入 Cookie），以极低的延迟（约 100ms）完成鉴权页面的全流程自动化点击、截图和验证，这是普通 AI 编程工具完全做不到的。
- **一人顶一个团队：** 它完美解决了单 AI 代理"既要写代码又要测代码"的角色混乱问题。通过并行 Sprint，你可以让规划、开发、测试同时进行，极大压缩单人开发的周期。

#### 劣势/局限

- **领域垂直：** 它几乎完全是为软件研发（尤其是 Web 产品从 0 到 1 的阶段）设计的。如果你只是想做市场营销方案或者写几篇小红书笔记，gstack 的 20 多个工程角色就显得过于沉重且不对口。
- **流程较重：** 对于极其简单的修修补补或个人小脚本，走一遍完整的 gstack 流程可能会觉得繁琐。

### 3.2 gstack 的 7 步标准化工作流（Sprint）

#### 第一阶段：产品需求与规划

- **技能指令：** `/office-hours` 或 `/autoplan`
- **核心作用：** AI 会化身 YC 投资合伙人或 CEO，通过核心问题倒逼你厘清真实痛点，推翻伪需求，重构产品定位，并输出可落地的完整设计文档。

#### 第二阶段：产品设计与架构

- **技能指令：** `/plan-design-review`、`/plan-eng-review`
- **核心作用：** 资深设计师会评估你的 UI/UX 设计，检测 AI 生成的"设计废料"；工程经理则会输出数据流、状态机、异常路径的完整架构图，把所有隐藏的技术坑提前暴露。

#### 第三阶段：研发编码与实现

- **技能指令：** 配合 Claude Code 进行代码编写，期间可使用 `/design-consultation` 等技能辅助。
- **核心作用：** gstack 的核心设计哲学是"拒绝半成品"。它强制要求每个环节都做到 100% 完备，哪怕是一个微小的功能模块，也要覆盖全链路的校验。

#### 第四阶段：测试与质量保障

- **技能指令：** `/review`、`/qa`、`/benchmark`
- **核心作用：**
  - **代码审查：** 化身资深高级工程师，专揪能通过 CI 但会在生产环境崩溃的深层 Bug（如竞态条件等高危风险）。
  - **真实浏览器测试：** 这是 gstack 的硬核突破。它在后台常驻浏览器进程，AI 与浏览器的每一次点击、输入、截图延迟被压缩到 100ms，还能完美保留登录态（直接导入 Chrome/Arc 的 Cookie），完成鉴权页面的全流程自动化测试。

#### 第五阶段：上线与复盘

- **技能指令：** `/ship`、`/retro`
- **核心作用：** 自动同步主分支、跑测试、审计、推送代码并开启 PR。上线后，还可以召开 AI 复盘会议，记录这个 Sprint 的经验教训，下次不再踩同样的坑。

### 3.3 Agency Agents：跨行业的"特种部队"

#### 优势

- **全行业角色覆盖：** 它不仅仅局限于写代码，而是提供了一个包含 144 个专业 AI 员工的庞大库，覆盖设计、工程、市场、质量、运营等 12 大部门。无论你是需要"增长黑客"制定裂变计划，还是需要"小红书运营"写爆款笔记，都能找到对应的人格化专家。
- **人格化与深度：** 每个 Agent 都有自己独特的沟通风格、工作方式和交付标准。比如"Senior Developer"不仅会写代码，还内置了常见的架构模式和避坑策略；"Reddit Community Ninja"懂得如何像一个真实用户一样去社区做运营，而不是生硬地发广告。
- **轻量灵活：** 它本质上是一个纯 Markdown 存储的角色规格库，非常轻量。你可以只挑选当前任务最需要的 3-5 个 Agent 激活，也可以根据自己的需求微调提示词，甚至自定义新增角色。

#### 劣势/局限

- **缺乏强制流程：** 它提供的是"专家"，但没有像 gstack 那样强制的"SOP 流程"。如果你不主动协调，可能会出现专家之间上下文传递断档的情况（比如产品经理的输出需要你手动完整传给前端开发）。
- **执行依赖宿主：** 它本身不是运行框架，必须依赖 Claude Code、Cursor 等宿主工具来执行具体任务。
- **成本与状态控制：** 在多 Agent 频繁协作的长链条任务中，Token 消耗会呈指数级增长；且当 Agent 数量过多时，保持全局状态的一致性（记忆偏差）目前仍是一个挑战。

## 四、总结与建议

| 维度 | gstack | Agency Agents |
|------|--------|---------------|
| 定位 | 软件研发 SOP 平台 | 跨行业 Agent 角色库 |
| 适用人群 | 技术开发者 | 独立创作者、运营人员 |
| 优势 | 强制流程、真浏览器测试 | 角色丰富、轻量灵活 |
| 劣势 | 领域垂直、流程较重 | 缺乏强制流程、Token 成本高 |

- **如果你是技术出身**，正在从 0 到 1 开发一款 Web 产品或 App，并且对代码质量、测试覆盖率和上线流程有极高要求，**gstack** 是你的不二之选。它能帮你把工程流程做到极致规范。
- **如果你是独立创作者、运营人员**，或者你的工作涉及大量跨领域协作（比如既要写代码，又要做市场调研、写内容、搞社区运营），**Agency Agents** 会给你带来更大的惊喜。它能让你一个人就拥有一整支跨行业的专家顾问团。
